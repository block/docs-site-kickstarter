Documentation layouts help organize your content in a clear and consistent way. This template provides several pre-built layouts that you can use for different types of documentation pages.

## Available Layouts

### Page Layouts

#### 1. Default Documentation Layout
The default documentation layout includes:
- Left sidebar navigation
- Right-side table of contents
- Main content area
- Responsive design that adapts to different screen sizes

```jsx
---
layout: DocsLayout
title: My Documentation Page
---

```

#### 2. API Reference Layout
Optimized for API documentation with:
- Code examples
- Parameter tables
- Response examples
- Status codes

```jsx
---
layout: APILayout
title: API Reference
---

# API Documentation
```

#### 3. Tutorial Layout
Perfect for step-by-step guides:
- Progress indicator
- Code snippets
- Interactive examples
- Previous/Next navigation

```jsx
---
layout: TutorialLayout
title: Getting Started
---

# Tutorial Content
```

#### 4. Landing Layout
For documentation home pages and section introductions:
- Hero section
- Feature highlights
- Quick navigation cards
- Search integration

```jsx
---
layout: LandingLayout
title: Welcome to Docs
---

# Welcome Section
```

### Content Layout Components

These modular layout components can be used within any page layout to create rich, engaging content sections:

#### 1. [Bento Grid](/docs-site-kickstarter/docs/layouts/bento-grid)
A flexible grid system for showcasing features and content in an organized, visually appealing manner.
- 12-column responsive grid system
- Multiple item sizes (small, medium, large, tall)
- Interactive elements with hover effects
- Perfect for feature showcases and content organization

#### 2. [Header Section](/docs-site-kickstarter/docs/layouts/header-section)
A versatile header section component for creating engaging page introductions.
- Optional taglines, headlines, and descriptions
- Multiple size options (sm, md, lg, xl)
- Text alignment options (left, center, right)
- Light and dark variants
- Fully responsive design

#### 3. [Feature Section](/docs-site-kickstarter/docs/layouts/feature-section)
Versatile section component for highlighting features, benefits, and key information.
- Multiple layouts (split, stacked, centered)
- Image support with hover effects
- Responsive design with mobile optimization
- Great for product features and service explanations

#### 4. [Footer](/docs-site-kickstarter/docs/layouts/footer)
Comprehensive footer component with navigation, social links, and newsletter signup.
- Multiple variants (default, minimal, detailed)
- Navigation sections with organized links
- Social media integration
- Optional newsletter signup form

#### 5. [CTA Section](/docs-site-kickstarter/docs/layouts/cta-section)
Powerful call-to-action component designed to drive conversions.
- Multiple visual variants (default, dark, gradient, bordered)
- Flexible layouts (centered, split)
- Primary and secondary action buttons
- Custom content slots for additional elements

#### 6. [Blog Section](/docs-site-kickstarter/docs/layouts/blog-section)
Flexible blog section component for displaying articles and posts.
- Multiple layouts (grid, list, featured)
- Rich metadata support (categories, authors, dates, read times)
- Responsive design with customizable columns
- Perfect for content-heavy documentation sites

#### 7. [FAQ Section](/docs-site-kickstarter/docs/layouts/faq-section)
Interactive FAQ section with accordion functionality.
- Smooth expand/collapse animations
- Multiple layouts (single column, two-column)
- Visual variants (default, bordered, minimal)
- Full keyboard accessibility support

#### 8. [Logo Section](/docs-site-kickstarter/docs/layouts/logo-section)
Showcase company logos and build trust with flexible display options.
- Multiple layouts (grid, marquee, centered)
- Visual variants (default, grayscale, bordered)
- Interactive elements with hover effects
- Perfect for client showcases and partner displays

## Customizing Layouts

You can customize any layout by modifying the layout components in `src/layouts/`. Each layout is built using shadcn/ui components and can be styled using Tailwind CSS.

### Example: Modifying the Sidebar

```tsx
// src/layouts/DocsLayout.astro
---
import Sidebar from '../components/docs/DocsSidebar.astro';
---

<div class="docs-container">
  <Sidebar className="custom-sidebar" />
  <main>
    <slot />
  </main>
</div>
```

### Layout Configuration

Layouts can be configured through frontmatter in your MDX files:

```yaml
---
layout: DocsLayout
title: Page Title
description: Page description
slug: "index"
sidebar: true
toc: true
pagination: true
---
```

## Responsive Design

All layouts are fully responsive and adapt to different screen sizes:

- **Desktop**: Full navigation with sidebar and table of contents
- **Tablet**: Collapsible sidebar, inline table of contents
- **Mobile**: Mobile-optimized navigation with drawer menu

## Best Practices

1. **Choose the Right Layout**
   - Use the default docs layout for most documentation pages
   - Use the API layout for endpoint documentation
   - Use the tutorial layout for step-by-step guides
   - Use the landing layout for section introductions

2. **Consistent Navigation**
   - Keep navigation structure consistent across sections
   - Use breadcrumbs for deep nested pages
   - Provide clear previous/next links

3. **Content Organization**
   - Use clear headings and subheadings
   - Include a table of contents for longer pages
   - Group related content in sections

4. **Responsive Considerations**
   - Test layouts on different devices
   - Ensure content is readable on all screen sizes
   - Optimize images and media for mobile

## Custom Layouts

You can create custom layouts by:

1. Creating a new layout component in `src/layouts/`
2. Extending existing layouts with additional components
3. Modifying the layout styles using Tailwind CSS

Example of a custom layout:

```tsx
// src/layouts/CustomDocsLayout.astro
---
import BaseLayout from './BaseLayout.astro';
import CustomSidebar from '../components/CustomSidebar.astro';
import CustomHeader from '../components/CustomHeader.astro';

const { title, description } = Astro.props;
---

<BaseLayout title={title} description={description}>
  <div class="custom-layout">
    <CustomHeader />
    <div class="content-wrapper">
      <CustomSidebar />
      <main>
        <slot />
      </main>
    </div>
  </div>
</BaseLayout>
```
